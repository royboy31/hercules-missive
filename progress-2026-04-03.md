# Progress - 2026-04-03

## 1. "Missive uniquement" (Missive-Only Products) Feature

**Goal:** Products marked as "Missive only" should be hidden from the public website but remain accessible in the Missive CRM for quotes.

### Components Implemented:

**A. WordPress mu-plugin (`hercules-missive-only.php`)**
- Exposes `missive_only` field in WooCommerce REST API
- Filters missive-only products from WooCommerce frontend queries (shop, search, categories)
- Uses existing ACF field `_missive_only` (no duplicate checkbox)

**Files deployed:**
- `combel-fr:/var/.../wp-content/mu-plugins/hercules-missive-only.php`
- `combel:/var/.../wp-content/mu-plugins/hercules-missive-only.php`
- `combel-uk:/var/.../wp-content/mu-plugins/hercules-missive-only.php`

**B. Product Sync Worker** (already implemented)
- `/products` endpoint filters out `missive_only: true` products
- Individual product endpoint supports `?exclude_missive=true` parameter
- Astro builds only include non-missive products

**C. Edge Router (`hercules-edge-router-fr-prod`)**
- Added runtime check for `/products/[slug]/` requests
- Calls product sync worker with `?exclude_missive=true`
- Returns 404 if product is missive-only or not found
- Prevents direct URL access to hidden products

```typescript
// Added to workers/edge-router/src/index.ts
const productMatch = pathname.match(/^\/products\/([^/]+)\/?$/);
if (productMatch && !isWordPress) {
  const checkUrl = `${env.PRODUCT_SYNC_WORKER_URL}/product/${productSlug}?exclude_missive=true`;
  const checkResp = await fetch(checkUrl);
  if (!checkResp.ok) {
    return new Response('...', { status: 404 });
  }
}
```

### How It Works:
| Location | Behavior |
|----------|----------|
| WooCommerce shop/search | Hidden (mu-plugin filters queries) |
| Astro /boutique page | Hidden (product sync filters index) |
| Astro /products/[slug] | 404 (edge router blocks at runtime) |
| Missive CRM | Visible (no filtering applied) |
| REST API | Visible (for CRM quote creation) |

## 2. Fixed Addon/Attribute Duplication in Quotes

**Issue:** Attributes showing twice with different labels:
```
Qualité maillot running    Qualité Pro   (from WooCommerce variation)
Choisez votre qualité      Qualité Pro   (from CRM selections)
```

**Root Cause:** CRM prompt labels differ from WooCommerce attribute names:
- WooCommerce: `"Qualité maillot running"`
- CRM prompt: `"Choisez votre qualité"`

Original filter only checked if keys matched - they didn't, so both were displayed.

**Fix:** Added **value-based duplicate detection**:
```php
// Method 3: Same value = duplicate (most reliable)
if ( $val_lower === $attr['value'] ) {
    $is_duplicate = true;
    break;
}
```

**Full duplicate detection now uses 3 methods:**
1. Exact label match: `key == label`
2. Partial key match: key contains/is-contained-in label or base name
3. **Value match**: selection value == attribute value (NEW)
