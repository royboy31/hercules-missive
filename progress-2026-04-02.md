## 2. Fixed "Addon" Hardcoded Label in Email/PDF Templates

**Issue:** Addon options showing "Addon" prefix instead of actual names like "Format", "Couleur".

**Root Cause:** PHP template used `get_term($addon_id, 'product_addons')` which failed for string keys from CRM selections.

**Fix:** Check if key is numeric (term ID to look up) or string (use directly as name):
```php
if (is_numeric($addon_id)) {
    $addon_term = get_term($addon_id, 'product_addons');
    $addon_name = $addon_term && !is_wp_error($addon_term) ? $addon_term->name : 'Addon';
} else {
    $addon_name = $addon_id; // Key is already the addon name
}
```

## 3. Fixed Duplicate Attribute Display (Format showing twice)

**Issue:** Quote #18860 showed "Format" and "Choisissez le format" both displaying same value.

**Root Cause:**
1. WooCommerce variation attributes displayed from `$variation->get_attributes()`
2. CRM selections (used as addons fallback) also contained the same attribute

**Fix:** Updated `hercules-crm-quote-api.php` to filter out selections that match variation attributes:
```php
// Get attribute labels and base names
foreach ($var_product->get_attributes() as $attr_key => $val) {
    $attr_labels[] = strtolower(wc_attribute_label($clean_key, $parent));
    $attr_base_names[] = strtolower(str_replace('pa_', '', $clean_key));
}
// Skip selections matching attributes
foreach ($item['selections'] as $key => $val) {
    if (!in_array(strtolower($key), $attr_labels, true)) {
        // Also check if key contains base name (e.g., "Choisissez le format" contains "format")
        $is_attr = false;
        foreach ($attr_base_names as $base) {
            if (strpos(strtolower($key), $base) !== false) {
                $is_attr = true;
                break;
            }
        }
        if (!$is_attr) {
            $addons[$key] = $val;
        }
    }
}
```

## 5. UK VAT Exemption - Complete Immunity

**Issue:** UK was showing 0% VAT when customer had VAT number, even before region was selected.

**Fix:** UK now **never** gets VAT exemption:
```tsx
const baseVatPercent = (() => {
  if (!selectedRegion) return 20; // No region: default 20%
  if (selectedRegion === 'UK') return FALLBACK_VAT['UK'] || 20; // UK: always 20%
  return dynamicVatRate !== null ? dynamicVatRate : (FALLBACK_VAT[selectedRegion] || 20);
})();

const isVatExempt = (() => {
  if (!selectedRegion) return false; // No region: never exempt
  if (selectedRegion === 'UK') return false; // UK: NEVER exempt
  // DE/FR logic unchanged...
})();
```

**VAT Rules:**
| Region | VAT Rate | Exemption |
|--------|----------|-----------|
| No region | 20% | Never |
| UK | 20% (always) | Never (ignores VAT number) |
| DE | dynamic/19% | If any VAT number |
| FR | dynamic/20% | If valid FR VAT format |

## 8. Comparison Table with Addon Prices

**Issue:** Comparison table showed base prices (e.g., 3.90) instead of prices with addons (e.g., 6.68).

**Example:** Casquette trucker with "Badge sublimé" + "Patch sublimation 60 cm2" showed 3.90 in table but actual price was 6.68.

**Fix:** Calculate addon prices at each tier quantity:
```tsx
conditionalPrices: (matchedVariation.conditional_prices || []).map((cp: any) => {
  const tierQty = parseFloatSafe(cp.qty);
  const basePrice = parseFloatSafe(cp.price);
  let addonPrice = 0;
  for (const addon of visibleAddons) {
    if (selectedAddons[addon.id]) {
      addonPrice += getAddonPriceAtTierQty(addon, selectedAddons[addon.id], tierQty);
    }
  }
  return { qty: tierQty, price: Math.round((basePrice + addonPrice) * 100) / 100 };
}),
```
